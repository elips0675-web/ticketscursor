2. Express 4 → 5 (следующий спринт)
Breaking changes, которые коснутся проекта:
Async errors в middleware: Express 5 не падает при throw в async middleware. Нужно проверить, что центральный error-handling middleware всё ещё ловит ошибки.
Путь роутов: Express 5 строже с regex в путях.
План миграции:
bash
npm install express@^5.0.0
cd server && npm test        # прогнать 138 тестов
npm run type-check           # проверить типы
node check-console.mjs       # 6 страниц, errors: []
Если тесты падают — не мержить, оставить в отдельной ветке.
3. ESLint warnings → 0 (можно параллельно)
Проблема: no-unused-vars ~100 штук. Скорее всего это:
Неиспользуемые импорты из shadcn/ui
Аргументы функций, оставленные «на будущее»
Деструктуризация с неиспользуемыми полями
Решение:
bash
npm run lint -- --fix        # автофикс импортов
Для осознанных неиспользуемых аргументов — префикс _:
TypeScript
// Было
function handler(req, res, next) { /* next не используется */ }

// Стало
function handler(req, res, _next) { }
Или настроить правило:
JavaScript
// eslint.config.js
'@typescript-eslint/no-unused-vars': ['warn', { 
  argsIgnorePattern: '^_', 
  varsIgnorePattern: '^_' 
}]
После чистки — снизить --max-warnings с 100 до 20 в CI, чтобы не разъезжалось.
4. Virtualizer в Tickets.tsx (отложенно, если планируется рост)

Интеграция — заменить <table> на виртуализированный контейнер с estimateSize. Но это только если метрики покажут лаги.
