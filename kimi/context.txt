## Стек

## Модули

- **Тикеты** — CRUD, SLA (due_at/first_response/resolved), автоназначение, валидация статусов, чат внутри
- **Real-time** — Socket.IO + Redis adapter, rate-limit, эмиты ticket/chat/user
- **RBAC** — 5 ролей (super_admin > admin > senior_agent > agent > requester)
- **Поиск** — Meilisearch → MySQL FULLTEXT → LIKE fallback, Ctrl+K
- **Файлы** — multer + magic bytes + ClamAV, S3/MinIO adapter
- **Wiki / Новости / Календарь / Опросы** — полноценные CRUD
- **Уведомления** — In-app + Email (nodemailer) + Telegram + Web Push
- **Аудит** — Все действия в audit_log

## Аутентификация

- JWT access (15min) + refresh httpOnly cookie (7d)
- LDAP/AD (ldapjs, auto-provisioning)
- Rate-limit: auth 10/min, api 100/min, admin 30/min

## Архитектура

- express.Router() → services → Prisma (чистая архитектура)
- Тонкие контроллеры, единый ответ `{ success, data, message }`
- RBAC через middleware `requireRole()`
- Error-handling через центральный middleware
- auditLogMiddleware на каждом защищённом роутере

## Правила работы
1. Код на русском (UI), комментарии по минимуму
2. Перед изменениями читать существующий код, повторять стиль
3. Не ломать работающее — сначала понять, потом менять
4. Все изменения через `git add -A && git commit && git push`
5. После задач запускать `npm test` и `npm run build`
6. После каждого изменения — запускать `node check-console.mjs` (errors: [], hasRussianText: true)

## Автоматический контроль

| Этап | Инструмент |

| Pre-commit | Husky → lint-staged (eslint --fix + prettier) |
| CI (GA) | tsc --noEmit → eslint --max-warnings 100 → vitest → vite build |
| Ручной | npm run lint / type-check / test / check:console |

# TECHNICAL.md (ключевые разделы)

**Архитектура:** routes → services → Prisma. Тонкие контроллеры, единый формат ответа, централизованный error-handling, RBAC middleware.

**SLA:** due_at/first_response_at/resolved_at, пересчёт при смене приоритета, фоновая проверка 15 мин, per-user dedup.

**Real-time:** Socket.IO + Redis adapter, rate-limit (5 msg/sec, 10 handshake/min per IP), эмиты ticket/chat/user.

**Тесты:** Vitest + jsdom (client 209, 44 файла), Vitest + supertest (server 295, 20 файлов), Playwright (E2E 22).
