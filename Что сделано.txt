Что сделано — Service Desk
============================

Стек: React 19 + TypeScript + Vite 8 + Tailwind v4 + shadcn/ui + Express + Prisma + MySQL 8 + Socket.IO + Redis

────────────────────────────────────────────────────────────────────────────

МОДУЛИ

  ✅ Аутентификация и RBAC
    • JWT access (15min) + refresh httpOnly cookie (7d), rate-limit 10/min
    • Пять ролей: super_admin > admin > senior_agent > agent > requester
    • requireRole() middleware, ProtectedRoute, скрытие UI по роли
    • LDAP (ldapjs, auto-provisioning), forgot/reset password (nodemailer)
    • Logout endpoint — инвалидация refresh token в БД

  ✅ Тикеты
    • CRUD, статусы с валидацией переходов (VALID_TRANSITIONS), приоритеты
    • SLA: due_at, first_response_at, resolved_at; пересчёт при смене приоритета
    • Автоназначение (AUTO_ASSIGN — наименее загруженный)
    • Agent видит свои + неназначенные; requester — только свои
    • Чат внутри тикета + внутренние заметки, вложения
    • WebSocket эмиты, фоновая проверка SLA (15 мин), экспорт CSV/PDF
    • POST /:id/messages — проверка прав (creator/assignee/senior_agent+)
    • Фильтры, пагинация, сортировка
    • ticket-context.tsx: хардкод localhost:4000 → централизованный API_URL

  ✅ Дашборд
    • Статистика, график по статусам (Recharts), виджет сотрудников
    • Последние обновления, клик по карточкам → переход с фильтром

  ✅ Чаты (WebSocket)
    • Общие и личные чаты, real-time сообщения, пагинация (50/запрос)
    • typing indicator, онлайн-статус, реакции
    • WebSocket rate-limit (5/сек, exponential backoff)
    • "Нельзя написать себе", макс 2000 символов

  ✅ Wiki / Новости / Календарь / Опросы
    • CRUD, RBAC (senior_agent+ для создания), пагинация
    • Загрузка изображений в wiki, CSV экспорт
    • Ближайшие события (виджет), прогресс-бары опросов

  ✅ Файловое хранилище
    • Drag & drop, папки, multer + magic bytes (file-type) + ClamAV
    • DELETE с проверкой роли (senior_agent+)
    • S3/MinIO адаптер (storage.js), fallback локальный диск

  ✅ Глобальный поиск
    • Meilisearch → MySQL FULLTEXT → LIKE fallback
    • 6 таблиц, парам. запросы (без SQL injection), Ctrl+K

  ✅ Сотрудники
    • Список (карточки/таблица), онлайн-статус, фильтры, CSV экспорт

  ✅ Админ-панель
    • Дашборд, управление пользователями (роли, блокировка)
    • Self-demotion запрещён, super_admin недоступен для выбора
    • Настройки интеграций, аудит действий, push-уведомления

  ✅ Уведомления
    • In-app (Socket.IO + БД) + Email (nodemailer) + Telegram
    • Автотриггеры: создание/статус/приоритет/назначение/сообщение/SLA
    • SLA per-user dedup — notifySlaBreached не блокирует алерты assignee/admin
