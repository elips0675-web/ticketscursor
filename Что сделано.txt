Контекст
Есть существующий проект Service Desk (Helpdesk) — корпоративная система учёта заявок с чатами, календарём, опросами, файловым хранилищем и профилем пользователя.
Текущий стек:
Frontend: React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui + Recharts
Backend: Express + MySQL + JWT (helmet, rate-limit)
Среда: Node.js 18+, MySQL (через Laragon)
Текущие модули: Дашборд, Тикеты, Сотрудники, Календарь, Опросы, Файлы, Чаты, Профиль, Авторизация
Известные проблемы: нет ORM, нет WebSocket/real-time, нет RBAC, нет Docker, нет тестов, dev-логин без пароля (бэкдор), файлы хранятся локально, нет валидации входных данных, нет refresh-токенов

Задача
Превратить проект в production-ready систему. Работать поэтапно, каждый этап — отдельный коммит/PR.

Этап 1. Инфраструктура и безопасность (критично)

  1. Заменить ручные SQL-запросы на Prisma ORM с полной типобезопасностью
  2. Настроить миграции базы данных через Prisma Migrate
  3. Добавить Zod для валидации всех входных данных на API
  4. Реализовать JWT с refresh-токенами (access — 15 мин, refresh — 7 дней, хранить в httpOnly cookie)
  5. Удалить dev-login эндпоинт из продакшена (оставить только под флагом NODE_ENV=development)
  6. Добавить RBAC (роли: super_admin, admin, manager, agent, user) с middleware-проверкой на каждом защищённом роуте
  7. Настроить CORS строго под домены приложения
  8. Добавить winston для логирования (в файл + консоль, ротация логов)
Инфраструктура:
  9. Создать docker-compose.yml с сервисами: app (Node), db (MySQL 8), redis (для сессий/кэша)
  10. Добавить .env.example со всеми обязательными переменными
