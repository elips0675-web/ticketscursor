🐛 Баг №1 (Критический): Хардкод localhost в ticket-context.tsx
Файл: src/context/ticket-context.tsx:8
Проблема: Захардкожен http://localhost:4000/api, хотя в проекте есть централизованный API_URL из @/lib/api.ts. В production фронтенд будет пытаться стучаться на localhost, что полностью сломает работу с тикетами.

🐛 Баг №2 (Критический): Ранний return в notifySlaBreached блокирует уведомления
Файл: server/src/notify.js:176-187
Проблема: Если уведомление о просрочке SLA уже отправлено создателю тикета (в течение 24ч), функция делает return и не отправляет уведомления assignee и админам. Это значит, что при повторной проверке просроченного тикета ответственные лица перестают получать алерты.

🐛 Баг №3 (Безопасность): Отсутствие проверки прав в POST /:id/messages
Файл: server/src/routes/tickets.js:241-270
Проблема: Любой авторизованный пользователь (включая requester) может писать сообщения в любой тикет, даже если он не является его создателем или исполнителем. В GET /:id проверка прав есть, а в POST /:id/messages — нет.

🐛 Баг №4 (Race Condition): Logout может отправить запрос без токена
Файл: src/context/AuthContext.tsx:52-61
Проблема: В функции logout токен берётся из замыкания React state, но после setToken(null) состояние ещё не обновилось. Если пользователь вызывает logout несколько раз быстро, второй вызов может отправить запрос без токена.
